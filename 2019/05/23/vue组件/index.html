组件

  • 组件化开发其实就是把页面拆分成一块一块的,每一个块可以看成是一个组件

全局注册组件

  • A组件可以调用另一个B组件，使得B组件在A组件的template选项中

  • template : 模板

  • 如果页面结构复杂，template选项不好维护

  • 抽离tempate选项的内容，template标签应该放到和body以及scripr标签同级

局部注册组件

  • new Vue 内部的组件
  • 如果一个A组件内部包含了一个组件B，那么在A组件定义的时候注册一个B组件

父组件给子组件传值

  • 父组件在调用子组件的地方，添加一个自定义的属性，属性的值为你需要传递给子组件的值

  • 在子组件定义的地方，添加了一个props选项，它的值是一个自由定义属性组合而成的数组，
    通过自定的属性名就可以找到获取到父组件传递过来的值.

  • 父组件在调用子组建的地方，添加一个自定义的属性，属性的值为你需要传递给子组件的值，如果
    需要传递的值是变量，那么需要使用到绑定属性

  • 在子组件定义的地方,添加了一个props选项，它的值是一个自由定义属性组合而成的数组或者是一个对象，如果需要数据类型的校验，则使用对象，否则可以使用数组
  • 在子组件的模块中，通过自定义属性名就可以获取到父组件传递过来的值

子组件给父组件传值

  • 水往低处流，若向高出流，那么就必须付出代价

  • 子组件中，通过某一个事件，执行this.$emit(‘自定义的事件’,’需要传递的值’)

  • 在父组件调用子组件的地方,给它绑定了子组件中自定义的事件，事件的实现由父组件实现，注意绑定的事件不要加()
  • 在父组件中实现添加参数，参数就是传递过来的值
  • 子组件中，通过某一个事件，执行this.$emit(‘自定义的事件’,’需要传递的值’),在父组件调用子组件的地方，给它绑定了子组件中的自定义的事件,事件的实现由父组件实现，注意绑定事件中不要添加()，在父组件中实现添加参数，参数就是传递过来的值

非父子组件之间的传值

  • 将值给一个邮递员，让他代传
  • 先表明可以接受数据,bus.on(‘自定义事件’,function(val){}) 如果要发送数据bus.emit(‘自定义事件’,val)

动态组件

  • 如果每一个组件内部含有表单元素，默认在动态组件切换的时候，表单数据会有所丢失，可以使用
    包裹动态组件，避免组件的重新渲染(手机按home键并不是退出了系统的应用程序)
  • include
  • 只装载a和b
