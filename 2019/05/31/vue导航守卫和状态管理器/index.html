状态态管理器

  • 单向数据流


  • 上面这是一个理想化的状态，但是，
  • 当我们的应用遇到多个组件共享状态时，单向数据流的简洁性很容易被破坏：

  • 多个视图依赖于同一状态。

  • 购物车、提交订单、个人中心各个视图都依赖于 一个登录 的状态

  • 当前案例中使用的是本地存储解决的问题，还可以使用状态管理器解决问题

  • 来自不同视图的行为需要变更同一状态。

  • 注册成功即为登陆成功，保存了登陆的状态，那么登陆成功也需要保存登陆状态

  • 对于问题一，传参的方法对于多层嵌套的组件将会非常繁琐，并且对于兄弟组件间的状态传递无能为力。

  • 对于问题二，我们经常会采用父子组件直接引用或者通过事件来变更和同步状态的多份拷贝。以上的这些模式非常脆弱，通常会导致无法维护的代码。

  • （网上买电脑 actions异步操作—-等待） —- 异步操作交给了状态管理器 – 异步操纵等待电脑到手

  • 另一种解读办法 （去店里买电脑） —- 异步操作交给组件 —- 异步操作是去店里
    本案例中使用第二种

  • 在组件内部先执行获取结果（在登陆组件中先登陆成功，在注册组件中先注册成功），提交状态管理器中 唯一改变 数据的方法（mutations），在方法的内部改变初始的状态
  • cnpm / npm i vuex -S (本案例中无需安装，脚手架中包含了 src/store.js)
  • src/store.js

  • 因为在main.js中引入了store.js，并且作为根实例给了new Vue，所以全局范围了可以通过this.$store 访问到状态管理器

  • 登陆页面变更状态

  • Login.vue中登陆成功之后，修改状态

  • 同理注册页面变更状态

  • 在需要判断登陆注册的地方，把本地存储的判断信息换成状态管理器的
  • 在购物车中实现未登录跳转到登录，已登录继续执行
