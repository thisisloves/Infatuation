开源协定：

  • GPL,LGPL,BSD,Apache

BSD:

  1. 如果二次发布的产品中包含源代码，则在源代码中必须带有原来代码的BSD协定
  2. 如果二次发布产品是二进制格式的库或者程序，则必须在发布的文档或者版权申明中说明包含原来代码的BSD协定
  3. 不可以用开源代码的作者或者组织，以及原来产品的名字做推广

程序的组成部分

  • 组成：二进制程序文件，库文件，配置文件，帮助文件
  • 二进制，库文件：可执行文件
  • 库文件：不能独立执行，只能被调用时执行
  • 配置文件，帮助文件：可被查看其它内容的文件
  • 人机交互页面

LINUX命令

  • 发起–命令：请求内核将某个二进制程序运行为一个进程：
  • 程序—进程

  • 静态—动态（有生命周期）

  • 命令本身是一个可执行文件：二进制格式的文件，有可能会调用共享文件

  • 多数系统程序文件都一般放在：/bin, /sbin ,/usr/sbin ,/usr/local/bin ,/usr/local/sbin
  • 普通命令： /bin ,/usr/bin ,/usr/local/bin

  • 管理命令：/sbin ,/usr/sbin ,/usr/local/sbin

  • 共享库：/lib ,/lib64 ,/usr/lib, /usr/local/lib64,

  • 32位bits库：
  • /lib, /usr/lib
  • 64位bits库：
  • /lib64 , /usr/lib64

注意:并非所有的命令都有一个在某目录与之对应的可执行文件

命令

  • 由shell程序自带的命令：内置命令（builtin）
  • 独立的可执行程序文件，文件即为命令名:外部命令
  • shell程序是独立的程序，负责解析用户提供的命令
  • 环境变量：
  • PATH:从哪些路径查找用户键入的命令字符串所对应的命令文件：

  • /usr/lib64/qt-3.3/bin:/usr/local/sbin:/usr/local/bin:/usr/bin:/root/bin

  • 查找次序：自左而右：

  • 查看命令类型：

  • type COMMAND

  • OPTIONS:指定命令的运行特征：

  • 短选项： -c,例如 -l ,-h 可合并

  • 长选项： -word,例如 -help

  • ARGUMENTS:

  • 命令的作用对象：命令对什么生效

LInux常用命令

  1. 文件系统严格区分大小写
  2. 可以使用除 /以外的的任意字符
  3. 长度不能超过255
  4. 以.开头的文件为隐藏文件
  • .当前目录

  • ..当前目录的上一级

  • 工作目录 :working directory

  • 家目录 ：home

  • 常用命令： pwd :printing working directory :显示工作目录当前目录

  • cd：change directory :切换目录

  • cd~:切换回自己家目录
  • cd~ username:切换指定用户家目录

  • ls : list,列出指定目录下的内容

  • ls [OPTIION]…..[FILE]…

  • a:显示所有文件，包括隐藏文件
  • A：显示除.和..之外的内容
  • l: long ,显示文件详细属性信息
  • h: —human–readable:对文件的大小进行单位换算
  • d:查看目录本身而非内部的文件的列表

  • r:reverse，逆序显示

  • cat :concatenate :文件文本查看工具

  • cat /etc/fstab

  • cat /etc/passwd

  • cat [OPTION]….[FILE]…..

  • -n :给显示的文本行编号

  • -E :显示行结束符$;

  • tac: 文本查看工具

  • tac …[OPTION]…[FILE]..
  • -n :给显示的文件行编号

  • -E ：显示行结束符

  • file :查看文件内容类型

  • file[FILE]….

  • echo: 回显

  • echo [SHORT-OPTION]….[STRING]
  • -n :不让换行

  • -e:让转义符生效

  • shutdown:关机或者重启

  • half -h

  • reboot -r

  • shutdown [OPTIONS…] [TIME] [WALL….]

  • 日期相关命令：date

  • cal： 日历

  • cal [[month] year]
