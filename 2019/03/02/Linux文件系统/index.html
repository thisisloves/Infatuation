linux文件系统

  • Filesystem

  • purpose ：目的

  • /bin:所有用户的基本命令程序文件 binaries:二进制 essential user command binaries 供所有的用户使用的基本命令程序文件

  • /sbin :供系统管理使用的工具程序仅root用户使用
  • /boot（鞋）:引导加载器必须使用到的各种静态文件：kernel,initramfs(initrd),grub等；
  • /dev:存储特殊文件或者设备文件
  • 设备有两种
  1. 字符设备（线性设备）
  2. 块设备（随机设备）
  • /etc ：系统程序的配置文件，只能为静态
  • host specific system configuration 主机特定系统配置文件
  • /home 普通目录的集中位置：一般每个普通用户的目录默认为此目录下与用户名同名的子目录，/home/Username:
  • /lib:（libraries图书馆） 为系统启动或者根文件系统的应用程序（/bin, /sbin等）提供共享库，以及为内核提供内核模块
  • lib.so.动态链接的C库
  • id* ：运行时链接器/加载器：
  • modules:用于存储内核模块的目录
  • /lib64:64位系统特有的存放64位共享库的路径
  • /media:便携式设备挂载点，cdrom,flop
  • /mnt:其它文件系统的挂载点
  • /opt:附加应用程序的安装位置，可选路径
  • /srv:当前主机为服务提供的数据；

  • /tem:哪些会产生临时文件的目录；可供用户执行写入操作；有特殊权限；

  • /usr :usr Hierachy ,全局共享的只读路径

/usr文件系统

  • /usr 文件系统经常很大，因为所有程序安装在这里 　　
  • bin,sbin
  • lib,lib64
  • include :c程序头文件
  • share：命令手册和自带文档架构特有的文件存储位置
  • local:另一个层级目录
  • x11R6:X-window程序的安装位置
  • src:程序源码文件的存储位置
  • /usr/local:local hierachy,让系统管理员安装本地应用程序，也通常用于安装第三发程序：
  • /var ：/var Hierarchy,存储发生变化的数据的目录：

/var文件系统

  • /var 包括系统一般运行时要改变的数据.每个系统是特定的，即不通过网络与其他计算机共享.
  • cache（cache:高速缓存） Appalication cache data
  • lib Variable（变量） satate information （信息）
  • local Variable data for /usr/local
  • lock Lock（锁） files
  • log （log 日志）Log files and directories
  • opt Vaeiable data for /opt
  • run Data relevant to running processees
  • spool Application spool data

  • tmp （temproary 临时文件）Temprorary files preserved between system reboots

  • /proce :内核及进程存储其相关信息:他们多为内核参数，例如net .ipv4.ip_forward,虚拟为net/ipv4/ip_forward,存储于/proc/sys/,因此- 其完整路径为/proc/sys/net/ipv4/ip.forward:

  • /sys:sysfs虚拟文件系统提供了一种比proc更为理想的访问内核数据的途径：其主要的作用在于管理Linux设备提供一种比proc 更为理想的访问内 核数据的途径：其主要作用在于管理Linux设备提供统一模型的接口。

linux系统文件类型

  • 常规文件
  • d:directory 目录文件
  • b:block device 快设备文件
  • c:character device ,字符设备文件，支持以”character”为单位进行线性访问
  • major number :主设备号，用于标识设备类型，进而确定要加载的驱动程序
  • minor number : 次设备号，用于标识同一类型中的不同的设备
  • 8位二进制：0-255
  • l：symbolic link 符号链接文件
  • p:pipe:命名管道：
  • s:socker，套接文件:

linux文件管理工具

  • cp 命令：copy

  • 源文件：目标文件

  • 单源复制:cp[OPTION]…….[-T]sourece dest
  • 多源复制:cp[OPTION]…..SOURECE……DIRECTORY
    cp [option]........-t directory  source
  • 单源复制 cp [option]……[-T] SOURCE DEST
  • 如果dest不存在，则事先创建此文件，并复制源文件的数据到dest中
  • 如果dest存在
  • 如果dest是非目录文件，则覆盖目标文件
  • 如果dest是目标文件，则先在dest目录下创建一个与源文件同名的文件，并复制其数据流
  • 多源复制 cp [OPTION]…..SOURCE… DIRECTORY
  • cp [OPTION]…..-t DIRECTORY SOURCE…
  • 如果dest不存在：错误
  • 如果dest存在
  • 如果dest是非目录文件：错误

  • 如果dest是目录文件：分别复制每个文件至目标目录中，并保存原名

  • -a: |-dR —preserve all
