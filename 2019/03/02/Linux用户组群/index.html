bash的基础特征

  • globbing :文件名通配符

  • 匹配模式：元字符

  • *：匹配任意长度的任意字符

  • pa,pa,pa,pa*

  • pa paa passwd

  • ?:匹配任意单个字符

  • 例如：a?
  • 可以匹配开头为a仅有2个字符的字符串 ab ac ad
  • 有几种特殊格式
  • 显示/var目录下所有以l开头，以一个小写字母结尾，且中间出现一位任意字符的文件或者目录
  • 复制/etc目录下，所有以m开头，以非数字结尾的文件或者目录至/tem/magedu.com目录

zip命令

  • zip命令
  • 功能：将多个文件归档压缩为zip文件
  • unzip命令
  • 功能：解压扩展名为.zip的文件
  • rpm 命令
  • 功能：查询已经安装的软件包
  • yum 命令
  • yum首先与源文件进行连接，并且自动选择传输速度最快的镜像网站，然后建立安装过程

ps命令

  • 功能：显示进程的状态。无选项的时候显示当前用户在当前终端启动的进程
  • 主要选项说明：
  • -a :显示当前终端上的所有进程
  • -A:显示系统所有进程，包括其他进程和系统进程信息
  • -l:显示进程详细信息，包括父进程号，进程优先级等

  • -u：显示进程详细信息cpu 内存使用等等

  • eg.使用 -l选项显示当前进程的详细信息

  • 主要输出项说明
  • S 进程状态 ，其中R表示运行状态；S表示休眠状态 T表示暂停 Z表示僵死状态
  • UID:进程所有者的用户ID
  • PID：进程号
  • PPID：父进程号
  • NI:进程的优先级
  • SZ:进程所占用的内存空间大小 ，以KB为单位
  • TTY:终端号
  • TIME：进程启动时间

  • CMD：启动该进程的SHell命令

  • 使用u选项显示当前进程的详细信息

  • 主要输入项说明
  • %CPU CPU的使用率
  • %MEM 内存的使用率
  • VSZ 占用虚拟内存的大小

  • STAT 内存状态

  • top命令

  • 功能：动态显示cpu的利用率，内存利用率和进程状态等相关信息。

mount命令

  • 格式：mount [选项] [设备名] [目录]
  • 功能：查看文件系统挂载情况或者将磁盘设备挂载到指定的目录
  • eg: 挂载光盘
  • 将/dev/cdrom的iso9660挂载到 /media/cd目录下
  • eg.挂载U盘
  • 将/dev/sad1 挂载到media/usb目录下
  • ummount 卸载磁盘

  • eg.ummount /dev/sad1

  • chattr 只有root用户才可以使用该命令的权限，

  • 作用修改文件系统的属性。

Io重定项以及管道

  • 程序：指令+数据
  • 程序： IO
  • 可用于输入的设备：文件

  • 键盘设备，文件系统上的常规文件，网卡等

  • 可用于输出的设备，文件

  • 显示器，文件系统上的常规文件，网卡等

  • 程序的数据流有三种

  • 输入的数据流 ：<– 标准输入（stdin）,键盘：
  • 输出的数据流：–>标准输出（stdout）,显示器

  • 错误输出流: –> 错误输出（stderr）,显示器

  • fd:file descriptor,文件描述符

  • 标准输入 ：0
  • 标准输出： 1

  • 错误输出：2

  • IO重定项：

  • 输出重定向>

  • 特性：覆盖输出
  • 输出重定向>>

  • 特征：追加输出

  • #set -c

  • 禁止覆盖输出重定向至已经存在的文件

  • 此时可以使用强制覆盖输出： >

  • #set +c

  • 关闭上述特征

  • 错误输出流重定项：2>,2>>

  • 合并正常输出流和错误输出流：
    (1) &>.$>>
    (2) COMMAND>/path/to/somefile 2>&1
    COMMAND>/path/to/somefile 2>&1

  • 输入重定向：<

  • tr命令：

  • tr [OPTION]….SEFI[SEF2]

  • 把输入的数据当中的字符，凡是在SET1定义范围内出现的，通通对位转换为SET2出现的字符

  • 用法1：

  • tr SET1 SET2< /PATH/FROM/SOMEFTLE
  • 用法2：
  • tr -d SET1 </PATH/FROM/SOMEFILE

  • 注意不修改原文件

  • Here Document: <<

  • cat <<EOF
  • cat >PATH/TO/SOMEFILE <<EOF
  • 管道：连接程序，实现将前一个命令的输出直接定向后一个程序当做输入数据流
  • COMMAND1 |COMMAND2 |COMMAND3 |………….
  • 例如：利用管道统计当前目录中的文件和子目录的数目
  • ls |wc -l
  • result :10

  • 屏幕上并没有显示ls的执行结果，这是因为ls命令通过管道交给wc -l来执行，屏幕最后显示wc -l的执行结果

  • tee命令

  • COMMAND1 | tee /PATH/TO/SOMEFILE

  • 把/etc/passwd文件的前六行的信息转换为大写字符后输出

  • head -n 6/etc/passwd |tr ‘a-z’ ‘A-Z’

linux用户组群

  • 用户类别：
  • 管理员，普通用户（系统用户和登陆用户）
  • 组类别：
  • 管理员组，普通用户组
  • 基本组和附加组

  • 私有组和公共组

  • 用户id(UID)组群id(GID)

  • etc/passwd :密码之外的账户信息

  • root:x:0:0:root:/root:/bin/bash

  • 用户名 ：密码：用户id:组群id:全名：用户主目录：登陆shell

  • /etc/shadow 用户密码信息文件

  • login:encrpted_password….

  • /etc/group 组群账号信息文件

  • group:password:gid:user:user_list

  • 组群名：组群加密密码：组群管理员密码：组群附加用户列表

  • groupadd:添加组

  • groupadd [选项] group-name

  • g:GID 修改GID

  • n:new 修改组群名

  • groupdel命令：删除组

  • groupdel[选项] group-name

  • useradd:添加用户

  • -u,—–uid：UID 指定uid

  • -g,—–gid:GROUP 指定基本组id
  • -G,—-groups GROUP1[,GROUP2,GROUP3]指明用户所属附加组，多个组之间用逗号隔开
  • -c,—-comment Comment:指明注释信息
  • -d,—–home HOME_DIR: 以指定的目录为家目录
  • -s,—–shell SHELL指定用户的默认shell

  • -r, 创建系统用户

  • useradd -D:显示创建用户的默认配置

  • userad -D:选项修改默认选项的值
  • usermod： 修改用户属性
  • usermod [选项]登陆
  • -u :修改用户的IO为此处指定的新的UID
  • -g: gid 修改用户基本组
  • -G: 修改用户附加组
  • -a:—-append:与G一起使用，用于为用户组追加新的附加组
  • -c:—comment 修改注释信息
  • -d: —home 修改用户家目录
  • -m: —-移动家目录
  • -l:修改用户名
  • -s:—–Shell 修改用户默认的shell
  • -l： —lock 锁定用户密码

  • -u ： —-unlock 解锁用户密码

  • userdel：删除用户

  • userdel[选项]登陆

  • -r :删除用户时一并删除其家目录

  • passwd命令：

  • passwd:修改用户自己的密码

  • passwd USERNAEM：修改指定用户的密码，仅root有此权限

  • change命令：更改用户密码过期信息

  • chage [选项] 登陆名
  • -d
  • -e
  • -w
  • -m
  • -M
  • id命令：显示用户的真和有效ID
  • id[OPTION]….[USER]
  • -u:仅显示有效的UID
  • -g:仅显示用户的基本组ID

  • -G:仅显示用户所属组的所有

  • su: switch user

  • 登陆切换：会通过读取目标用户的配置文件来重新初始化
